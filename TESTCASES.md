Тест-кейсы для API микросервиса объявлений

Позитивные тест-кейсы

TC001: Создание объявления
Endpoint: POST /api/1/item
Предусловия: Нет
Шаги:
1. Отправить POST запрос с валидными данными объявления
2. Проверить статус код ответа
3. Проверить структуру ответа
Ожидаемый результат:
- Статус код: 200
- В ответе присутствует поле "status"
- Объявление успешно создано

TC002: Получение объявления по ID
Endpoint: GET /api/1/item/{id}
Предусловия: Создано тестовое объявление
Шаги:
1. Создать тестовое объявление
2. Получить ID созданного объявления
3. Отправить GET запрос по полученному ID
4. Проверить статус код ответа
5. Проверить структуру ответа
Ожидаемый результат:
- Статус код: 200
- Ответ является списком
- Данные объявления возвращены корректно

TC003: Получение всех объявлений продавца
Endpoint: GET /api/1/{sellerID}/item
Предусловия: Создано несколько объявлений для одного продавца
Шаги:
1. Создать 2 объявления с одинаковым sellerID
2. Отправить GET запрос для получения всех объявлений продавца
3. Проверить статус код ответа
4. Проверить структуру ответа
Ожидаемый результат:
- Статус код: 200
- Ответ является списком
- Возвращены все созданные объявления
