Отчет о багах

Найденные проблемы

Bug 1: Несоответствие формата ответа при создании объявления
Эндпоинт: POST /api/1/item
Описание: При создании объявления возвращается строка статуса вместо структурированного JSON с ID объявления
Ожидаемое поведение: 
{
  "id": "uuid",
  "sellerId": 123,
  "name": "Test Item",
  "price": 1000,
  "statistics": {...},
  "createdAt": "timestamp"
}
Фактическое поведение:
{
  "status": "Сохранили объявление - uuid"
}
Серьезность: Minor
Влияние: Усложняет парсинг ответа и получение ID объявления

Bug 2: Отсутствие валидации входных данных
Эндпоинт: POST /api/1/item
Описание: API принимает отрицательные значения цены и статистики без ошибки валидации
Шаги воспроизведения:
1. Отправить запрос с price: -100
2. Проверить ответ
Ожидаемый результат: Ошибка валидации (400 Bad Request)
Фактический результат: Объявление создается успешно
Серьезность: Major
Влияние: Возможность создания некорректных данных
